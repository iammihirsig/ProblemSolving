# ProblemSolving

Welcome to the **ProblemSolving** repository! This repository is a comprehensive collection of solved problems from coding fundamentals to advanced Data Structures and Algorithms (DSA) from various competitive programming platforms. It is designed to serve as a single point of reference to revisit, revise, and practice problems or review previous solutions.

## About

Hello! My name is **Mihir Raj Singh**, and I am the author of this repository. The main purpose of this repository is to **organize** and **track my progress** in solving problems from **fundamentals to advanced DSA**.

I have added required documentation as comments in each code file. **Also, note that I will be focusing on brute force solutions and how to move towards more optimal approaches.**

This repository acts as a personal resource for revision and a way to share solutions with the programming community.

## File Naming Convention

All files follow a consistent naming convention for better organization and readability:

```
SerialNo_DifficultyLevel_TitleOfProblem.cpp
```

### Example

- `1_E_Even_Odd.cpp`
  - **1**: Serial number
  - **E**: Difficulty level (E = Easy, M = Medium, H = Hard)
  - **Even_Odd**: Title of the problem

## How to Use

This repository is structured to help you effectively revise or learn DSA problems.

1. **Navigation**

   - Browse the sequentially organized `.cpp` files to revisit solutions and explanations.

2. **Comparison of Approaches**

   - Read through different solution approaches for each problem.
   - Compare their **time complexity (TC)** and **space complexity (SC)** to decide which approach is the best for a given problem.

3. **Practice**
   - Solve the problems independently before checking the solutions to reinforce your understanding.
   - Use the solutions to validate your approach or learn alternative methods.

## Getting Started

To start using this repository:

1. Clone the repository to your local machine:

   ```bash
   git clone https://github.com/iammihirsig/ProblemSolving/
   ```

2. Navigate to the specific file or folder of interest:

   ```bash
   cd ProblemSolving
   ```

3. Open the `.cpp` file of the problem you wish to explore.

4. Try solving the problem on your own before referring to the **Solutions** section for guidance.
