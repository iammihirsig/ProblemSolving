Welcome to the **ProblemSolving** repository! This repository is a comprehensive collection of solved problems in Data Structures and Algorithms (DSA) from various competitive programming platforms. It is designed to serve as a single point of reference to revisit, revise, and practice problems or review previous solutions.

## About

Hello! My name is Mihir Raj Singh, and I am the author of this repository. The main purpose of this repository is to organize and track my progress in solving problems related to DSA.

Each problem solved is carefully documented, with the **problem link**, and **solutions**, provided for different approaches, such as brute force, optimized algorithms, or best practices.

This repository acts as a personal resource for revision and a way to share solutions with the programming community.

## File Naming Convention

All files follow a consistent naming convention for better organization and readability:
```
SerialNo_DifficultyLevel_TitleOfProblem.md
```

### Example
- `1_E_Contains_Duplicate.md`
  - **1**: Serial number
  - **E**: Difficulty level (E = Easy, M = Medium, H = Hard)
  - **Contains_Duplicate**: Title of the problem


## Repository Structure

Each file contains the following sections:
1. **Problem Links**
   - Links to the problem on various platforms (e.g., Leetcode, Neetcode, ).
   - The platform doesn’t matter as long as the problem is freely available to solve on the internet.

2. **Solutions**
   - Different approaches to solve the problem, including:
     - **Brute Force**
     - **Optimized Techniques**
     - **Best Approach**
   - Each approach includes code written in **C++** and an analysis of its **time complexity** and **space complexity**.

## How to Use

This repository is structured to help you effectively revise or learn DSA problems.

1. **Navigation**
   - Browse the sequentially organized `.md` files to revisit solutions and explanations.

2. **Comparison of Approaches**
   - Read through different solution approaches for each problem.
   - Compare their **time complexity** and **space complexity** to decide which approach is the best for a given problem.

3. **Practice**
   - Solve the problems independently before checking the solutions to reinforce your understanding.
   - Use the solutions to validate your approach or learn alternative methods.

## Getting Started

To start using this repository:

1. Clone the repository to your local machine:
   ```bash
   git clone https://github.com/iammihirsig/ProblemSolving/
   ```

2. Navigate to the specific file or folder of interest:
   ```bash
   cd ProblemSolving
   ```

3. Open the `.md` file of the problem you wish to explore.

4. Use the links provided in the **Problem Links** section to access the problem online.

5. Try solving the problem on your own before referring to the **Solutions** section for guidance.
